# When To Use

Use this skill after implementing a screen or flow, especially before sharing, recording, or extending the demo.

# Review Focus

Check for:

- visual drift from Figma
- weak spacing or hierarchy
- generic-looking styling that breaks the intended taste
- inconsistent navigation behavior
- static areas that should feel interactive
- repeated UI that should be extracted
- comments or TODOs that are missing where logic is unclear

# Review Heuristics

- Prefer findings that affect the perceived quality of the demo.
- Call out places where the app feels like a mockup instead of a believable product.
- Suggest the smallest changes that create the biggest polish gains.
- Keep recommendations grounded in this repo's current stack and structure.

# Expected Output

Return:

1. the highest-impact polish issues first
2. any interaction gaps in the target user flow
3. quick wins for improving presentation quality
